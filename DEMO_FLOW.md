# 웹 데모 사용 흐름

## 🎯 핵심 기능 시연 순서

### 1단계: 프로젝트 생성
1. 메인 화면에서 **"새 프로젝트 만들기"** 클릭
2. 제목 입력: "테스트 소설"
3. 장르 선택: "소설 — 문학"
4. **"만들기"** 클릭
5. ✅ 자동으로 교열 탭으로 이동
6. ✅ **왼쪽 사이드바에 "설정집", "커스텀 사전" 메뉴 나타남**

---

### 2단계: 교열 실행 (핵심!)
1. 교열 탭에서 원고 입력:
   ```
   그는 공원에 앉아서 하늘을보았다. 구름이 천천히 움직이고있었다. 마음이 복잡했다. 결정을해야했다.
   ```

2. **"교열 시작"** 클릭

3. 1.5초 후 결과 표시:
   - **교열된 원고** 탭: 수정된 텍스트
   - **수정 제안 (3)** 탭: 3개 제안 목록
   - **설정집** 탭: 인물/장소 정보 (교열 결과 내)

4. ✅ 토스트 메시지: **"교열 완료 - 설정집 및 커스텀 사전 업데이트됨"**

---

### 3단계: 설정집 확인 ⭐ 
1. **왼쪽 사이드바 → "설정집" 클릭**

2. 확인 내용:
   - **인물 프로필**: "주인공 - 공원에서 고민하는 인물"
   - **장소 / 공간**: "공원 - 주인공이 결정을 고민하는 장소"
   - 나머지 섹션: "등록된 항목 없음"

3. ✅ **교열 결과가 자동으로 프로젝트에 저장됨!**

---

### 4단계: 커스텀 사전 확인 ⭐
1. **설정집 화면에서 상단 탭 → "커스텀 사전" 클릭**

2. 확인 내용:
   - **서사 상태 변화**:
     - [인물] 주인공 등장 - 공원에 앉아 있음
     - [모순] 결정을 내려야 하는 상황 (구체적 내용 미상)
   - **교열 이력**:
     - 날짜, 글자 수, 수정 제안 개수

3. ✅ **시간순 추적 가능!**

---

### 5단계: 추가 교열 테스트
1. 다시 **교열 탭** 클릭

2. 새로운 원고 입력:
   ```
   주인공은 이름이 민수다. 그는 회사원이다. 공원에서 그는 친구 영희를 만났다.
   ```

3. **"교열 시작"** 클릭

4. **설정집 → 인물 프로필** 확인:
   - ✅ 기존: "주인공"
   - ✅ 새로 추가: "민수", "영희"
   - ✅ **중복 없이 누적됨!**

---

## 💡 주요 차별점

### ❌ 일반 AI 도구
- 교열 결과만 보여주고 끝
- 원고 전체 일관성 체크 불가
- 이전 교열 내용 기억 안 함

### ✅ Edito 웹 데모
- **설정집 자동 생성** - 인물/장소/용어 추출
- **커스텀 사전** - 서사 상태 변화 추적
- **프로젝트별 저장** - 여러 번 교열해도 누적
- **왼쪽 사이드바에서 언제든 확인 가능**

## 🎬 데모 시연 스크립트 (1분)

```
안녕하세요. Edito는 장편 원고의 일관성을 체크하는 AI 편집 도구입니다.

[프로젝트 생성]
먼저 프로젝트를 만들고...

[교열 실행]
원고를 입력하고 교열을 실행하면,
단순히 교정만 하는 게 아니라,

[설정집 클릭]
왼쪽 사이드바에 설정집이 자동으로 생성됩니다.
보시는 것처럼 인물, 장소 정보가 자동으로 추출됐죠.

[커스텀 사전 클릭]
커스텀 사전에는 서사 상태 변화가 기록됩니다.
이렇게 쌓인 정보로, 300페이지 원고도 일관성을 유지할 수 있습니다.

[다시 교열]
추가로 교열하면, 정보가 계속 누적되고,
1장에서 "검은 눈동자"였다가 50장에서 "푸른 눈동자"가 되는
모순을 자동으로 잡아낼 수 있습니다.

이것이 Edito의 핵심 차별점입니다.
```

## 📸 스크린샷 권장

1. **프로젝트 생성 모달**
2. **교열 탭 - 입력 화면**
3. **교열 결과 - 수정 제안**
4. ⭐ **설정집 화면 - 인물/장소 자동 추출**
5. ⭐ **커스텀 사전 - 서사 상태 변화**
6. **사이드바 - 설정집/커스텀 사전 메뉴**

---

## 🐛 예상 질문 & 답변

**Q: 설정집 메뉴가 안 보여요**
A: 프로젝트를 먼저 만드셔야 합니다. 프로젝트 없이는 설정집 메뉴가 숨겨져 있습니다.

**Q: 설정집이 비어있어요**
A: 교열을 실행해야 자동으로 채워집니다. 교열 전에는 빈 상태가 정상입니다.

**Q: 실제 앱과 다른 점은?**
A: 웹 데모는 모킹 데이터를 사용합니다. 실제 앱은 EXAONE 3.0 LLM이 원고를 분석해서 설정집을 추출합니다.

**Q: 프로젝트를 여러 개 만들면?**
A: 각 프로젝트마다 독립된 설정집/커스텀 사전을 가집니다.
